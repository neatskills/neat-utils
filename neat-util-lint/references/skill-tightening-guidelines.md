# Skill Tightening Guidelines

Phase 3 condensing rules.

## Target: 30-50% reduction (best-effort)

- **Already concise** (< 200 words or no clear reduction): Skip with note
- **Some tightening** (< 30%): Apply what helps clarity
- **30-50% achievable**: Apply target range

Clarity over percentage. Guideline, not requirement.

## Preserve

- Workflow steps, order
- Checks, validations, conditions
- Section structure, order
- Diagrams, tables, code
- Examples (different concepts only)
- Frontmatter, `**Role:**`

## Tighten

1. Remove filler: "In order to", "It is important", "You should", "Make sure"
2. Shorten descriptions
3. Lists over paragraphs
4. Combine redundant sentences
5. One example per concept
6. Direct voice ("Run X" not "X should be run")

## Red flags (STOP)

- Removing/combining workflow steps
- Simplifying conditional logic
- Changing section structure/order
- Removing different-scenario examples
- Changing meaning
- Removing *why* (keep *why*, remove *how* if obvious)
- Converting tables ↔ prose
